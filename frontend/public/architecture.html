2. Core Components & Design Choices

  • Frontend (React/Vite): Captures audio natively and downsamples it to 16kHz PCM chunks using an AudioWorkletNode. Offloads processing to ensure a responsive UI.
  • Real-Time Gateway (Socket.io/Node.js): Low-latency bidirectional bridge between browser and backend, allowing instantaneous pushing of transcripts without REST polling.
  • Event-Driven Backbone (Kafka in KRaft mode): Decouples system components via distinct topics (raw-audio, transcripts, insights). Ensures variable LLM response times never block ingestion.
  • Local STT (Vosk): Locally hosted STT engine via WebSocket cuts latency and cloud costs.
  • Multi-Agent AI (Gemini 1.5 Flash): Parallelizes AI tasks via contextual routing triggered by keywords, optimizing token usage.
  • Persistence (PostgreSQL/Prisma): Securely preserves call sessions, transcripts, and AI insights.

3. Real-Time Data Flow (Sequence)

sequenceDiagram
    participant Browser
    participant Backend as Backend (Socket.io)
    participant Kafka
    participant Vosk as Vosk STT
    participant Agents as AI Agents
    participant Gemini

    Browser->>Backend: start-call { sessionId, title }
    Backend-->>Browser: session-started { sessionId }

    loop Audio Streaming
        Browser->>Backend: audio-chunk { sessionId, chunk }
        Backend->>Kafka: produce → raw-audio
        Kafka->>Backend: consume (audio-processors)
        Backend->>Vosk: stream PCM bytes
        Vosk-->>Backend: { text: "final transcript" }
        Backend->>Kafka: produce → transcripts
        Kafka->>Backend: consume (transcript-feed)
        Backend-->>Browser: transcript
    end

    par Agent Processing
        Kafka->>Agents: consume transcripts
        Agents->>Gemini: generateContent()
        Gemini-->>Agents: insight text
        Agents->>Kafka: produce → agent-insights
        Kafka->>Backend: consume (result-aggregators)
        Backend-->>Browser: insight
    end
