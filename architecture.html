Wingman System Architecture

The underlying architecture of Wingman is designed as an event-driven, real-time streaming system optimized for low-latency audio processing and scalable AI inference.

Core Components & Design Choices

  • Frontend (React/Vite): Captures audio natively and downsamples it to 16kHz PCM chunks using an AudioWorkletNode. Offloads audio processing to ensure a responsive UI.
  • Real-Time Gateway (Socket.io/Node.js): Low-latency bidirectional bridge between browser and backend, allowing instantaneous pushing of transcripts without REST POLLING.
  • Event-Driven Backbone (Kafka): Decouples system components via distinct topics (raw-audio, transcripts, insights). Ensures variable LLM response times never block ingestion.
  • Local STT (Vosk): Locally hosted STT engine via WebSocket cuts latency and cloud costs.
  • Multi-Agent AI (Gemini 1.5): Parallelizes AI tasks via contextual routing (Sales Coach, Search) triggered by keywords, optimizing token usage.
  • Persistence (PostgreSQL): Preserves call sessions and insights securely.

Scalability Capabilities

  • Stateless Gateway Layer: Orchestrator nodes scale horizontally behind load balancers for concurrent streams.
  • Elastic STT Workers: Containerized STT engines scale independently.
  • Kafka Consumer Groups: Duplicate instances of any AI agent can be deployed linearly; Kafka automatically balances the load.
